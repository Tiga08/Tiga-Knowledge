# 截图/文本题库提取与答题

从培训考试截图或粘贴文本中识别题目,可整理为结构化知识文档,或基于已有知识文档作答。**无论输入来源与意图,最终都必须为每道题给出「答案 + 答案来源」。**

## 输入与意图

- **输入来源**:截图(用 Read 多模态识别,逐张处理)/ 粘贴文本(直接解析)。
- **意图**:
  - **A 整理文档**:把题目/解析整理为知识点,合并进目标 Markdown。
  - **B 作答**:基于已有知识文档为题目作答。
- **判定**:截图或粘贴的「解析/知识点文本」默认走 A;出现「答题 / 给答案 / 这些题选什么 / 帮我做这套题」等求解意图走 B;不明时主动询问。
- **目标文件**:A 需目标文件路径(通常在 `00-inbox/questions/`),B 需已有知识文档作引用来源;缺失则询问。

## 答案来源标注(两个工作流通用)

每道题答案都要标来源,取以下之一:

| 情况 | 标注 |
|------|------|
| 原题已含答案/解析 | `原题标注` |
| 知识文档命中 | `文件名:行号`(如 `2-中联重科辉煌征程.md:39`) |
| 外部查证 | `外部出处:{链接/资料名}` |
| 无法确定 | `(待确认,{依据})` |

> 行号随文档变动失效:每次作答前重新 Read 取当前行号,不复用历史行号;`文件名:行号` 引用必须真实存在于当前文档,否则改注外部出处,不得臆造行号。

## 工作流 A:提取并更新知识文档

1. **识别**:逐张截图 / 逐批粘贴文本,识别题型(单选/多选/判断)、题干、选项、已标答案、解析。
2. **取答案**:原题含答案则直接提取;否则用 WebSearch/WebFetch 查证并交叉验证多来源,无法确定标 `(待确认)`。
3. **整理知识点**:去选项保留核心知识点;正确答案写入描述;易混淆错误选项写为排除项(「注意:xxx 不属于 yyy」);按主题分章。单条出处不确定时,在该条下方用缩进嵌套 blockquote 注明依据。
4. **逐批写入**:每处理完一张截图/一批题目即写入,不等全部完成。首批先 Read 检查目标文件——不存在则 Write 创建(含 H1、来源 blockquote、章节);已存在则 Read 后 Edit 合并(去重、相同主题并入对应章节、新主题追加新章节并中文编号递增)。
5. **输出**:① 变更摘要(识别题数 / 新增知识点 / 更新章节 / 待确认项);② 带来源的答案表(见「答案输出格式」)。

## 工作流 B:基于文档作答

1. **实时 Read** 目标知识文档,取当前行号。
2. 逐题检索命中知识点确定答案;未命中用 WebSearch/WebFetch 查证;仍不确定标 `(待确认,{依据})`。
3. **输出**:带来源的答案表(见「答案输出格式」)。

## 答案输出格式(两个工作流通用)

先给带来源详表:

| 题号 | 答案 | 来源 |
|------|------|------|
| 1 | A | 2-中联重科辉煌征程.md:39 |
| 2 | 正确 | 原题标注 |
| 3 | ABD | 外部出处:{链接/资料名} |
| 4 | B | (待确认,常识判断) |

再给速查表(便于直接填写):

```
1. A
2. 正确
3. ABD
4. B
```

- 单选/多选输出选项字母(`A` / `ABD`);判断题输出 `正确` 或 `错误`;原题缺选项无法确定字母时输出文本答案并标 `(待确认)`。

## 知识文档格式模板

```markdown
# {主题}

> 整理自"{试卷名称}"(总分 X 分:单选 N 题 + 多选 N 题 + 判断 N 题),去除题目选项,仅保留核心知识点。

---

## 一、{章节主题}

- **{知识点关键词}**:{正确内容描述}。注意:{排除项/易混淆点}。

---

## 二、{下一章节主题}

- **{知识点}**:{描述}。
```

格式规则:

- **H1 = 纯主题名**(如 `# 做廉洁自律的中联人`),不带「知识汇总」等后缀。
- H1 下一条 blockquote 来源说明,题型与分值信息可选,信息不完整时只写已知部分。
- H2 用中文数字编号(一、二、三…);知识点用 `- **加粗关键词**:描述`,正确答案与排除项写在同一条目;章节间用 `---` 分隔。
- 单条出处不确定时,在条目下方用缩进嵌套 blockquote 注明依据:

```markdown
- **收购 WILBERT**:2018 年,100% 股权收购德国 WILBERT 塔式起重机制造商。
  > 基于中联重科并购公开报道综合判断
```

## 注意事项

- **去重**:写入前 Read 现有文件,避免重复知识点。
- **排除项标注**:易混淆错误选项必须写为排除说明,帮助记忆。
- **事实准确**:不编造答案,无法确认标 `(待确认)`。
- **脱敏**:截图与粘贴文本均需脱敏,不含个人信息、账号、IP、token 等。
