# Tiga-Knowledge

长期技术知识、学习笔记、论文阅读、工程经验与系统化认知沉淀。

---

## 目录结构

| 目录 | 说明 |
|------|------|
| `00-inbox/` | 临时收集区，未整理资料统一进入此处 |
| `01-computer-science/` | 计算机基础知识 |
| `02-ai/` | AI、CV、LLM、模型部署与推理优化 |
| `03-engineering/` | 工程实践、开发规范、调试部署 |
| `04-algorithms/` | 算法、数据结构、数学基础 |
| `05-papers/` | 论文阅读、论文笔记、复现记录 |
| `06-books/` | 书籍阅读笔记 |
| `07-courses/` | 课程学习记录 |
| `08-interview/` | 技术面试知识沉淀 |
| `09-english/` | 英文学习与技术英语 |
| `10-blogs/` | 技术博客草稿与发布归档 |
| `90-assets/` | 公共图片、截图、附件 |
| `91-templates/` | 通用笔记模板 |
| `92-references/` | 外部资料索引 |
| `99-archive/` | 过期或低频内容归档 |
| `agent-plan/` | Agent 生成的计划、审查报告、迁移方案 |

## 文件命名规范

- 普通文件：`kebab-case.md`
- 日期文件：`YYYY-MM-DD-title.md`
- 论文目录：`year-paper-short-name/`
- 课程目录：`course-name/`
- 图片附件：`assets/` 下按主题归档

## 维护节奏

- **每日：** 临时资料进入 `00-inbox/`
- **每周：** 整理 `00-inbox/`，移动到对应主题目录
- **每月：** 审查重复笔记、过期资料、空目录
- **每季度：** 沉淀专题索引、归档低频内容
- **每年：** 重构知识地图与学习路线
